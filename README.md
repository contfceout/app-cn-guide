# app-cn-guide

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态文件形式存放，旨在提供一种轻量的信息展示方式。仓库本身不针对任何特定域名或网站，仅作为 HTML 资源的集中存储与历史版本管理使用。

## 目录说明

```
.
├── index.html          # 仓库入口文件（可选）
├── pages/              # 存放主要 HTML 页面文件
│   ├── example1.html
│   └── example2.html
├── assets/             # 公共资源（CSS、JS、图片等）
│   ├── styles/
│   └── scripts/
└── README.md           # 本文件
```

- `pages/`：每个独立 HTML 页面建议放在此目录下，便于管理。
- `assets/`：存放页面共享的样式、脚本或图片资源，避免重复引用。

## 页面归档说明

- 所有 HTML 页面均为独立文件，可直接在浏览器中打开，或通过 GitHub Pages 等静态托管服务部署。
- 页面内容与外部资源（如 CSS、JS）尽量使用相对路径引用，以保证在本地或服务器上均能正常访问。
- 本仓库不保证页面内容的时效性或准确性，仅作为归档用途。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面或更新已有页面。
- 请确保新增页面不包含恶意代码、外部追踪脚本或违反平台政策的内容。
- 建议为每次页面更新附带清晰的提交信息，方便追溯变更历史。
- 如发现页面无法正常显示，请提交 Issue 说明问题。
